# 缓存与定时任务模块

## 一、Redis 缓存（TongRDS）

### 1.1 技术栈
- 中间件: TongRDS V2.2（100% 兼容 Redis 协议）
- 客户端: spring-boot-starter-data-redis (Lettuce)
- 配置: `config/RedisConfig.java` — RedisTemplate (Jackson + JavaTimeModule 序列化)
- 封装: `service/CacheService.java` — 所有操作统一 try-catch 降级

### 1.2 缓存策略汇总

| Key 模式 | 类型 | TTL | 用途 | 写入时机 | 失效时机 |
|----------|------|-----|------|---------|---------|
| `session:{userId}` | String(JSON) | 24h | 用户会话信息 | 拦截器首次查 DB 后 | 修改状态/角色时主动删除 |
| `event:quota:{eventId}` | String(int) | 24h | 报名配额原子计数 | 活动审批通过时 | 自动过期 |
| `event:slots:{eventId}` | String(int) | 10s | 剩余名额展示 | 查询活动列表时 | 报名/取消后主动删除 |
| `event:lock:{eventId}:{userId}` | String | 5s | 报名防重锁 | 报名请求开始时 | 请求结束后主动删除 |
| `club:views:YYYYMMDD` | ZSET | 8天 | 社团每日浏览量 | 调用浏览接口时 | 自动过期 |
| `club:hot:top20` | String(JSON) | 1h | 热门社团缓存 | 首次查询计算后 | 自动过期 |
| `config:approval:expense-threshold` | String | 无 | 经费审批阈值 | 管理员配置时 | 手动更新 |
| `config:approval:club-stages` | String(JSON) | 无 | 社团审批阶段 | 管理员配置时 | 手动更新 |

### 1.3 降级策略
- 所有 CacheService 读操作: try-catch → 返回 null → 业务层回退到 DB 查询
- 报名配额 DECR 失败: 回退到 `countActiveSignups` DB 查询（接受微量超卖风险）
- 防重锁获取失败: 降级为放行（`return true`）

### 1.4 CacheService 方法清单

**会话缓存**:
- `cacheSessionInfo(userId, info)` — 写入 24h
- `getSessionInfo(userId)` — 读取
- `evictSessionInfo(userId)` — 主动删除

**热门社团**:
- `incrementClubView(clubId)` — ZINCRBY 当日 ZSET
- `getTopViewedClubIds(topN)` — ZUNIONSTORE 7天 + ZREVRANGE
- `cacheHotClubs(clubs)` — 写入 1h
- `getHotClubs(typeRef)` — 读取

**名额展示**:
- `cacheEventSlots(eventId, remaining)` — 写入 10s
- `getEventSlots(eventId)` — 读取

**报名配额**:
- `initEventQuota(eventId, count)` — SET 24h
- `decrementEventQuota(eventId)` — DECR（原子）
- `incrementEventQuota(eventId)` — INCR（回滚用）
- `getEventQuota(eventId)` — 读取

**防重锁**:
- `trySignupLock(eventId, userId, timeoutMs)` — SETNX
- `releaseSignupLock(eventId, userId)` — DEL

**配置**:
- `setConfig(key, value)` — SET（无 TTL）
- `getConfig(key)` — GET

## 二、定时任务

### 2.1 调度器

#### EventStatusScheduler (`config/EventStatusScheduler.java`)
- **触发频率**: 每分钟 (`0 * * * * ?`)，启动时立即执行一次
- **功能**: 自动推进活动状态
  - 报名中(3) → 进行中(6): `start_at <= NOW()` 的活动
  - 进行中(6) → 已结束(4): `end_at <= NOW()` 的活动
- **SQL**: 直接通过 mapper 批量 UPDATE

#### UserLifecycleScheduler (`config/UserLifecycleScheduler.java`)
- **每天 2:00**: `GRADUATION_EXIT_CLUB` — 退出毕业生社团成员资格
- **每天 2:30**: `CLUB_CANCEL_FREEZE_ACCOUNT` — 冻结已注销社团管理员账号
- **每 10 分钟**: `AUTO_END_EVENTS` — 结束过期活动
- 通过 `ScheduledTaskService.executeBySystem()` 执行

### 2.2 任务状态机 (`ScheduledTaskServiceImpl`)

```
STOPPED ←→ STARTED → RUNNING → STARTED (成功)
                        ↓
                      FAILED
```

**状态说明**:
| 状态 | 含义 |
|------|------|
| STOPPED | 已停止，不会被调度执行 |
| STARTED | 已启动，等待下次调度 |
| RUNNING | 正在执行中 |
| FAILED | 上次执行失败 |

**命令操作**:
- `START`: STOPPED/FAILED → STARTED
- `STOP`: 非 RUNNING 状态 → STOPPED

**执行流程**:
```
ScheduledTaskService.executeBySystem(taskCode)
    ↓
检查状态 == STARTED (synchronized)
    ↓ 非 STARTED → 跳过
状态 → RUNNING
    ↓
执行 runTask(taskCode):
  GRADUATION_EXIT_CLUB → schoolAdminPermissionService.autoExitGraduates()
  CLUB_CANCEL_FREEZE_ACCOUNT → schoolAdminPermissionService.autoFreezeCanceledClubAdmins()
  AUTO_END_EVENTS → schoolAdminPermissionService.autoEndEvents()
    ↓
成功 → 状态回 STARTED, 记录 processedCount
失败 → 状态改 FAILED, 记录异常
```

**管理接口**（学校管理员）:
- `POST /api/school-admin/scheduled-tasks/{taskCode}/command` — 手动 START/STOP
- `GET /api/school-admin/scheduled-tasks/status` — 查询状态

## 三、基础设施工具

### 3.1 全局异常处理 (`GlobalExceptionHandler`)
- `BusinessException` → 返回自定义 code + message
- 参数校验异常 → 返回 1001 + 具体字段错误
- 其他异常 → 返回 1099 "系统异常"

### 3.2 统一响应格式 (`ApiResponse<T>`)
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

### 3.3 业务错误码 (`BizCode`)
| 代码 | 名称 | 说明 |
|------|------|------|
| 0 | SUCCESS | 成功 |
| 1001 | PARAM_INVALID | 参数错误 |
| 1002 | ACCOUNT_NOT_FOUND | 账号不存在 |
| 1003 | NO_PERMISSION / PASSWORD_INCORRECT | 无权限/密码错误 |
| 1004 | ACCOUNT_DISABLED / ACCOUNT_STATUS_INVALID | 账号不可用 |
| 1005 | ACCOUNT_EXISTS / RESOURCE_NOT_FOUND | 已存在/不存在 |
| 1006 | PASSWORD_WEAK / DATA_SCOPE_DENIED | 密码弱/越权 |
| 1007 | LOGIN_EXPIRED | 登录过期 |
| 1099 | SYSTEM_ERROR | 系统异常 |

### 3.4 ID 生成器 (`IdGenerator`)
- 雪花算法变体: `((timestamp - EPOCH) << 12) | sequence`
- Epoch: 2024-01-01 00:00:00 UTC
- 序列号: 12 位 (最大 4095)
- synchronized 保证线程安全

### 3.5 文件上传 (`CosUtil`)
- 腾讯云 COS 对象存储
- UUID 命名 + 安全后缀提取
- 懒加载 COS 客户端（synchronized）
- `@PreDestroy` 关闭客户端

### 3.6 分页包装 (`PageResponseData<T>`)
```json
{
  "records": [...],
  "total": 100,
  "pageNum": 1,
  "pageSize": 20
}
```
