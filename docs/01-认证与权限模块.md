# 认证与权限模块

## 一、模块概述

负责用户注册、登录、Token 管理、身份校验和角色权限控制。所有需要鉴权的 API 均通过拦截器统一校验 JWT Token。

## 二、API 端点

| 方法 | 路径 | 鉴权 | 说明 |
|------|------|------|------|
| POST | /api/auth/login | 否 | 用户名密码登录 |
| POST | /api/auth/register | 否 | 用户注册（默认 STUDENT 角色） |
| POST | /api/auth/sso/login | 否 | SSO 单点登录 |
| GET | /api/auth/me | 是 | 获取当前登录用户信息 |
| POST | /api/auth/refresh-token | 否 | 刷新 Access Token |
| POST | /api/auth/logout | 是 | 登出 |

## 三、核心实现逻辑

### 3.1 登录流程 (`AuthServiceImpl.login`)

```
用户提交 username + password
    ↓
trim(username) → 校验格式: ^[A-Za-z0-9_]{4,30}$
    ↓
查询用户: AuthMapper.findUserByUsername()
    ↓ 不存在 → ACCOUNT_NOT_FOUND
    ↓ 状态 ≠ 1 → ACCOUNT_DISABLED
比较密码: BCrypt.matches(password, passwordHash)
    ↓ 不匹配 → PASSWORD_INCORRECT
    ↓ 匹配 ↓
查询角色: AuthMapper.findPrimaryRoleCode(userId)
    ↓ null → 默认 STUDENT
查询社团ID: AuthMapper.findPrimaryClubId(userId, roleCode)
    ↓
生成 Access Token (有效期 7200s) + Refresh Token (有效期 7 天)
    ↓
更新 last_login_at
    ↓
返回 LoginResponseData(token, userId, username, realName, roleCode, status, clubId)
```

### 3.2 注册流程 (`AuthServiceImpl.register`)

```
校验 username 格式 + password 格式: ^(?=.*[A-Za-z])(?=.*\d).{8,64}$
    ↓
检查用户名是否已存在
    ↓ 存在 → ACCOUNT_EXISTS
生成雪花 ID → 创建 user 记录(status=1)
    ↓
查找 STUDENT 角色 ID（不存在则创建）
    ↓
创建 user_role 关联记录
    ↓
返回 RegisterResponseData(userId)
```

### 3.3 SSO 登录 (`AuthServiceImpl.ssoLogin`)

```
解析 ticket → 提取用户名（支持 "prefix:username" 格式）
    ↓
查找用户 → 不存在则自动注册（密码 SsoDefault123）
    ↓
走正常 buildLoginResponse 流程
```

### 3.4 Token 刷新 (`AuthServiceImpl.refreshToken`)

```
解析 Refresh Token → 验证 tokenType == "REFRESH"
    ↓
查询会话信息 → 验证用户状态为 ACTIVE
    ↓
生成新的 Access Token + Refresh Token
    ↓
返回 LoginResponseData
```

### 3.5 获取当前用户 (`AuthServiceImpl.me`)

```
从 ThreadLocal 获取 userId
    ↓
查询 AuthSessionInfo (用户基本信息 + 角色 + 社团ID)
    ↓
根据角色返回权限列表:
  STUDENT → [student:club:browse, student:club:join, student:event:signup]
  CLUB_ADMIN → [club:info:manage, club:event:manage, club:finance:manage]
  SCHOOL_ADMIN → [school:club:approval, school:finance:audit, school:data:stats, school:user-status:manage]
```

## 四、请求拦截器 (`AuthInterceptor`)

### 拦截范围
- 拦截: `/api/**`
- 排除: `/api/auth/login`, `/api/auth/register`, `/api/auth/sso/login`, `/api/auth/refresh-token`, `/error`

### 拦截逻辑
```
提取 Authorization 头 → "Bearer {token}"
    ↓ 缺失或格式错误 → LOGIN_EXPIRED
解析 JWT → 验证签名和过期时间
    ↓ 无效 → LOGIN_EXPIRED
验证 tokenType == "ACCESS"
    ↓
从 Redis 缓存获取 AuthSessionInfo (缓存 24h)
    ↓ 缓存未命中 → 查 DB → 写入缓存
    ↓ DB 也未找到 → LOGIN_EXPIRED
验证用户状态 == ACTIVE
    ↓ 非 ACTIVE → ACCOUNT_STATUS_INVALID
构建 CurrentUser → 存入 ThreadLocal (AuthContextHolder)
    ↓
请求处理完毕 → afterCompletion → 清除 ThreadLocal
```

## 五、JWT Token 结构

### Claims 字段
| 字段 | 类型 | 说明 |
|------|------|------|
| userId | Long | 用户 ID |
| username | String | 用户名 |
| permissionType | String | 角色代码 |
| tokenType | String | ACCESS 或 REFRESH |
| iat | Date | 签发时间 |
| exp | Date | 过期时间 |

### Token 有效期
- Access Token: 7200 秒（2 小时，可配置）
- Refresh Token: 604800 秒（7 天，硬编码）

### 签名算法
- HMAC-SHA，密钥来自 `app.jwt.secret` 配置

## 六、角色与权限体系

### 角色定义 (`role` 表)
| role_id | role_code | 说明 |
|---------|-----------|------|
| 1001 | STUDENT | 学生 |
| 1002 | CLUB_ADMIN | 社团管理员 |
| 1003 | SCHOOL_ADMIN | 学校管理员 |

### 权限检查工具 (`AccessChecker`)
- `requireLogin()` — 要求已登录，否则抛 LOGIN_EXPIRED
- `requireRole(String... roles)` — 要求角色在指定列表内，否则抛 NO_PERMISSION

### 用户状态 (`UserStatusUtil`)
| DB 值 | 状态文本 | 说明 |
|--------|---------|------|
| 1 | ACTIVE | 正常 |
| 2 | FROZEN | 冻结 |
| 3 | CANCELED | 注销 |
| — | GRADUATED | graduate_at <= 当前时间 |

## 七、会话缓存（Redis）

- Key: `session:{userId}`
- TTL: 24 小时
- 内容: `AuthSessionInfo` (userId, username, realName, status, graduateAt, roleCode, clubId)
- 写入时机: 拦截器首次从 DB 加载后
- 失效时机: 修改用户状态、修改角色、社团审批通过/注销时主动淘汰
