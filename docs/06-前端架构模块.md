## 三、HTTP 请求层 (`api/http.js`)

### Axios 配置
- baseURL: 环境变量 `VITE_API_BASE_URL`，默认 `/api`
- timeout: 10 秒

### 请求拦截器
```javascript
请求发出前:
  getAccessToken() → 存在则添加 Authorization: Bearer {token}
```

### 响应拦截器
```javascript
响应返回后:
  检查 response.data.code
    ↓ code !== 0 → 提取 message → reject(error)
    ↓ code === 0 → 返回 response
  ↓
HTTP 状态 401:
  clearAccessToken() → 跳转 /login?redirect={当前路径}
```

## 四、认证状态管理 (`stores/auth.js`)

### State
| 字段 | 类型 | 说明 |
|------|------|------|
| token | String | JWT Access Token |
| role | String | 标准化角色 (student/clubAdmin/schoolAdmin) |
| user | Object | 用户信息 (userId, username, realName, clubId 等) |

### Computed
| 属性 | 说明 |
|------|------|
| isAuthenticated | `!!token` |
| permissions | `getPermissionsByRole(role)` |
| userStatus | `user.status` 或从存储读取 |
| isUserActive | `userStatus === 'ACTIVE'` |

### Methods
| 方法 | 说明 |
|------|------|
| `bootstrap()` | 应用启动时从 localStorage/sessionStorage 恢复状态 |
| `setSession(data)` | 登录成功后存储 token、role、user（支持记住我） |
| `logout()` | 清除所有状态和存储 |

## 五、Token 存储 (`utils/token.js`)

### 存储策略
- **记住我** → `localStorage`（浏览器关闭后保留）
- **不记住** → `sessionStorage`（浏览器关闭后清除）
- 读取时: localStorage 优先于 sessionStorage

### 存储 Key
| Key | 说明 |
|-----|------|
| `club_access_token` | JWT Token |
| `club_user_role` | 用户角色 |
| `club_user_profile` | 用户信息(JSON) |

### 方法
- `getAccessToken()` / `setAccessToken(token, remember)`
- `getUserRole()` / `setUserRole(role, remember)`
- `getUserProfile()` / `setUserProfile(profile, remember)`
- `getUserStatus()` — 从 profile 中提取
- `clearAccessToken()` — 清除两种存储中所有 key

## 六、角色与权限 (`utils/role.js`)

### 角色映射 (`normalizeRole`)
后端返回多种形式（STUDENT, student, CLUB_ADMIN, clubAdmin 等），统一转为:
- `student`
- `clubAdmin`
- `schoolAdmin`

### 权限常量
| 角色 | 权限列表 |
|------|---------|
| student | student:club:browse, student:club:join, student:event:signup |
| clubAdmin | club:info:manage, club:event:manage, club:finance:manage |
| schoolAdmin | school:club:approval, school:finance:audit, school:data:stats, school:user-status:manage |

### 仪表盘路径
| 角色 | 路径 |
|------|------|
| student | /dashboard/student |
| clubAdmin | /dashboard/club-admin |
| schoolAdmin | /dashboard/school-admin |

## 七、路由系统 (`router/index.js`)

### 路由表
| 路径 | 组件 | 权限 |
|------|------|------|
| `/` | HomeView | 公开 |
| `/login` | LoginView | 公开 |
| `/register` | RegisterView | 公开 |
| `/dashboard` | — | 按角色重定向 |
| `/dashboard/student` | StudentDashboardView | student 角色 |
| `/dashboard/club-admin` | ClubAdminDashboardView | clubAdmin 角色 |
| `/dashboard/school-admin` | SchoolAdminDashboardView | schoolAdmin 角色 |
| `/unauthorized` | UnauthorizedView | 公开 |
| `/account-disabled` | AccountDisabledView | 公开 |

### 导航守卫逻辑
```
路由跳转前 (beforeEach):
    ↓
目标需要鉴权？(meta.requiresAuth)
    ↓ 否 → 放行
    ↓ 是 ↓
已登录？(authStore.isAuthenticated)
    ↓ 否 → 重定向 /login?redirect={目标路径}
    ↓ 是 ↓
用户状态 ACTIVE？
    ↓ 否 → 重定向 /account-disabled
    ↓ 是 ↓
角色匹配？(meta.role 与 authStore.role)
    ↓ 否 → 重定向 /unauthorized
    ↓ 是 ↓
权限满足？(meta.permission 在 authStore.permissions 中)
    ↓ 否 → 重定向 /unauthorized
    ↓ 是 → 放行

已登录用户访问 /login 或 /register:
    → 重定向到角色对应的仪表盘
```

## 九、应用启动 (`main.js`)

```javascript
1. 创建 Vue 应用实例 (App.vue)
2. 创建 Pinia Store
3. 安装 Pinia
4. authStore.bootstrap() — 从存储恢复登录状态
5. 安装 Vue Router
6. 挂载到 #app
```
