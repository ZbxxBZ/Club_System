# 学生端模块

## 一、模块概述

学生端提供社团浏览与入社、活动报名与签到、个人信息管理三大功能域。所有接口需登录，大部分限 STUDENT 角色。

## 三、核心功能逻辑

### 3.1 社团广场 (`listPublicClubs`)

**查询条件**：
- 仅显示 `status=2`（活跃）且已审批通过（`apply_status=4`）的社团
- 已加入且不在招新时间内的社团自动隐藏
- 支持关键字搜索（社团名称模糊匹配）、分类筛选、是否招新筛选

**每个社团计算字段**：
- `signedUpCount` = 在籍成员数 + 待审批入社申请数
- `joinStatus` = 当前用户的入社状态（JOINED/PENDING/REJECTED/NONE）
- `isRecruiting` = 招新状态为 OPEN 且当前时间在招新窗口内
- `joinEntryEnabled` = 综合判断用户能否申请加入（未加入、未申请、名额未满、正在招新）

### 3.2 申请加入社团 (`joinClub`)

```
要求 STUDENT 角色
    ↓
查询社团加入资格元数据:
  - 社团存在 且 status=2 且 已审批通过
  - 招新状态 OPEN 且 当前时间在招新窗口内
  - 在籍成员 + 待审批申请 < 招新名额(recruit_limit)
    ↓
检查用户是否已是成员 或 有待审批申请 → 是则拒绝
    ↓
插入 club_join_apply 记录 (join_status=1 待审批)
    ↓
返回申请ID
```

### 3.3 新建社团申请 (`submitClubApply`)

```
要求 STUDENT 角色
    ↓
检查用户是否已在任何社团（countActiveClubMemberships > 0 → 拒绝）
    ↓
检查社团名称是否重复
    ↓
创建 club 记录(status=1 筹建中) + club_apply 记录(apply_status=1, 步骤=MATERIAL_REVIEW)
    ↓
返回申请ID和社团ID
```

### 3.4 活动报名 (`signupEvent`) — 防超卖

```
获取分布式锁: SETNX event:lock:{eventId}:{userId} (5s TTL)
    ↓ 失败 → "请勿重复提交"
校验活动: 状态=3(报名中)、时间窗口、仅成员限制、重复报名
    ↓
Redis 原子扣减名额:
  DECR event:quota:{eventId}
    ↓ 结果 < 0 → INCR 回滚 + "名额已满"
    ↓ Redis 不可用 → 降级到 DB 查 countActiveSignups
    ↓ 结果 >= 0 → 继续
DB 写入:
  - 若有已取消记录 → 更新状态为 1(已报名)
  - 否则 → 插入新记录
    ↓ 唯一键冲突 → INCR 回滚 + "无法重复报名"
清除名额展示缓存 event:slots:{eventId}
    ↓
finally: 释放分布式锁
```

### 3.5 取消报名 (`cancelEventSignup`)

```
校验活动存在 且 状态非 6(进行中) 和 4(已结束)
    ↓
校验用户已报名
    ↓
更新报名状态为 2(已取消)
    ↓
INCR event:quota:{eventId} 归还名额
DELETE event:slots:{eventId} 清除展示缓存
```

### 3.6 活动签到 (`checkinByCode`)

```
要求 STUDENT 角色
    ↓
通过 eventId 或 checkinCode 定位活动
    ↓
校验:
  - 活动状态为 3(报名中) 或 6(进行中)
  - 当前时间在 startAt ~ endAt 之间
  - 用户已报名该活动
  - 用户未签到过
    ↓
尝试恢复已软删除的签到记录
  ↓ 恢复数=0 → 插入新签到记录
```

### 3.7 活动列表与名额缓存 (`listOpenEvents`)

```
查询 event 表 + 报名/签到子查询
  - 筛选: 状态 IN (3,6) 或 (用户已报名 且 状态 IN (3,4,6))
  - 可选: 关键字(标题/社团名)、只看已加入社团
    ↓
对每个活动:
  - canSignup = (eventStatus == 3)
  - 若有 limitCount:
    - 先查 Redis event:slots:{eventId} (TTL 10s)
    - 未命中 → countActiveSignups 计算 → 写入缓存
    - 设置 remainingSlots
```

### 3.8 热门社团 (`listHotClubs`)

```
查缓存 club:hot:top20 (TTL 1h)
    ↓ 命中 → 直接返回
    ↓ 未命中 ↓
ZUNIONSTORE 合并最近 7 天的 club:views:YYYYMMDD
    ↓
ZREVRANGE 0 19 → 取 TOP20 clubId
    ↓
mapper.listClubsByIds() → 批量查社团信息
    ↓
缓存结果 → 返回
```

### 3.9 个人信息管理

- **查看**: 从 `user` 表读取 username, realName, studentNo, phone, email
- **修改基本信息**: 更新 realName + studentNo
- **修改密码**: 旧密码验证 → 新密码格式校验(8-64位,含字母+数字) → BCrypt 加密存储
- **修改联系方式**: phone + email(邮箱格式校验)

### 3.10 文件上传 (`uploadClubApplyFile`)

**支持的 bizType**:
- CLUB_APPLY_CHARTER, CLUB_APPLY_INSTRUCTOR_PROOF — 社团申请材料
- CLUB_CANCEL_ASSET_SETTLEMENT — 社团注销资产清算
- EVENT_SAFETY_PLAN, EVENT_SUMMARY_ATTACHMENT — 活动相关
- EVENT_SUMMARY_IMAGE — 活动总结图片
- FINANCE_PROOF, EXPENSE_INVOICE — 财务凭证

**文件类型限制**:
- 图片类(EVENT_SUMMARY_IMAGE): jpg/jpeg/png/gif/bmp/webp
- 文档类(其他): pdf/doc/docx

**上传流程**: 校验 → CosUtil.upload() → 返回 COS 文件 URL
