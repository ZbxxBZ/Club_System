# 对话存档（截至 2026-03-11）

## 1. 项目定位与系统目标
本项目为“学校社团管理系统”前端（Vue3 + Vite），当前工作重点是：
- 用户与权限管理（学生 / 社团管理员 / 学校管理员）
- 学校管理员端用户治理（角色切换、账号禁用切换）
- 学校管理员端定时任务启停（按钮状态与后端状态联动）
- 与后端 API 文档对齐，便于联调

系统总体设计与需求已在以下文档沉淀：
- `docs/database-design.md`
- `docs/user-permission-module-api.md`
- `docs/auth-backend-requirements.md`

## 3. 当前 API 封装实现（前端）
文件：`src/api/user-permission.js`

### 3.1 学校管理员相关
- `getUserListApi(params)` -> `GET /school-admin/users`
- `updateUserStatusApi(userId, payload)` -> `PATCH /school-admin/users/{userId}/status`
- `disableUserApi(userId, reason)` -> 内部调用 `updateUserStatusApi(..., status=FROZEN)`
- `restoreUserApi(userId, reason)` -> 内部调用 `updateUserStatusApi(..., status=ACTIVE)`
- `updateUserRoleApi(userId, payload)` -> `PATCH /school-admin/users/{userId}/role`
- `getClubApprovalQueueApi(params)` -> `GET /school-admin/approvals/clubs`
- `decideClubApprovalApi(approvalId, payload)` -> `POST /school-admin/approvals/clubs/{approvalId}/decision`
- `getAuditStatsApi(params)` -> `GET /school-admin/statistics/overview`

### 3.2 定时任务相关
- `getScheduledTaskStatusApi(taskCodes)` -> `GET /school-admin/scheduled-tasks/status`
- `commandScheduledTaskApi(taskCode, payload)` -> `POST /school-admin/scheduled-tasks/{taskCode}/command`

任务编码约定：
- `GRADUATION_EXIT_CLUB`
- `CLUB_CANCEL_FREEZE_ACCOUNT`

命令约定：
- `START`
- `STOP`

## 5. 关键约束（必须遵守）

### 5.1 审计表约束
来自 `docs/database-design.md` 第 8 节：
- `audit_log` 仅用于社团审批业务审计（APPROVE/REJECT/RETURN）
- 非审批类功能（用户状态切换、定时任务启停、页面操作日志）**禁止写入 `audit_log`**
- 非审批留痕应写入独立日志表或统一操作日志系统

### 5.2 角色/状态展示依赖字段
前端已做兼容解析，但后端建议稳定返回：
- 角色：`roleCode`
- 状态：`status`

## 6. 已知风险与后端联调关注点
1. 角色切换成功但列表仍旧值
- 常见原因：主从延迟、缓存未失效、列表 SQL 未正确 join 角色关系

2. 状态切换按钮不按预期变色
- 常见原因：后端返回状态值不统一（建议标准化 `ACTIVE/FROZEN`）

3. 定时任务状态刷新后丢失
- 需确保 `GET /scheduled-tasks/status` 返回真实状态并与 taskCode 对齐
